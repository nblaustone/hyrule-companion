# knowledge — verified BotW data

The sourced, machine-checked data that feeds the app's reference tabs. **Don't hand-edit the clean files** —
they're regenerated by `build/assemble-knowledge.mjs`; edit the upstream or the assembler instead.

## Provenance
- `_raw-research.json` — the full output of the `botw-research` workflow (run `wf_426b04f1-8fb`, 41 agents,
  ~1.69M tokens): one research agent per the 15 Sheikah-Tower regions + global agents (armor, bestiary, cooking,
  koroks, world), **each adversarially verified** by a second agent, then a completeness audit. Includes the
  agents' `notes` (correction logs) — kept here for honesty/traceability, **stripped before the UI**.
- `regions.json`, `globals.json`, `audit.json` — the raw output split out (the audit flagged the desert/DLC
  problems; see the learning log).

## The clean, app-facing datasets (the assembler's output — reconciled 120 / 15 / 4)
- `shrines.json` — all **120** shrines, `{name, location, category, oneLine, shrineQuest, solution}`. Every shrine
  now carries a spoiler-gated **`solution`** (the exact how; for hidden ones, how to make it appear) — v12.7.
- `towers.json` (15) · `great-fairies.json` (4).
- `side-quests.json` — the **complete 78** base-game side quests (v12.11), each with a stable **`id`** (slug) +
  `giver/location/reward/oneLine/how` (spoiler-gated how-to). Shrine quests are derived from `shrines.json`, not here.
- `armor.json` (16 sets) — each upgradeable set has **`tiers`** (★1–★4 materials + rupees) + a **`farm`** note (v12.9).
- `bestiary.json` (34 enemies) — top-level **`basics`** (7-card combat primer) + a per-enemy **`battle`** guide on
  the 26 marquee fights (v12.8).
- `cooking.json` (reference) · `cooking-ingredients.json` (**120** sourced ingredients for the pot simulator, v10).
- `koroks.json` — enriched **puzzle types** (`{type, see, do, category}`) for the solver (v12.9).
- `economy.json` — rupee earners + material farming + money tips, powers Guide→Money (`EconomyView`, v12.9).
- `compendium.json` — the **410-entry item catalog** (`{name, cat, type, power?, durability?, effect?, where?,
  set?, sell?}`; cat ∈ weapon/bow/shield/armor/material/creature) for the Items tab + answer-first search (v12.13–14).
- `world.json` · `lore.json` (shared cross-game lore tales).
- `region-maps.json` — per-region schematic-map coordinates (shrines, tower, Great Fairy, landmarks) on a
  relative 0–100 grid (north-up), from the `botw-region-coords` workflow (15 agents, one per region; web-checked;
  clamped + de-overlapped at assembly — 0 gaps, 0 overlaps). Powers the `RegionMap` mini-maps in the Shrines tab.

## The content layer (v12.7+) — `build/gen-*-workflow.mjs` → `build/merge-*.mjs`
The deep reference fields above (`solution`, `battle`/`basics`, armor `tiers`, `economy`, korok enrichment, the
78 side quests, the 410-item compendium) were each authored by a **sourced author→adversarial-verify Workflow**
(web-checked against Game8/Zelda Dungeon/Thonky/Zeldapedia, every claim re-verified by an independent agent). A
matching `merge-*.mjs` splices ONLY the new field(s) into the clean JSON (additive; strips `sources`/`corrections`
so verification meta never reaches the UI). These bypass the 120/15/4 assembler — they extend existing records.

## How it was made correct (the honesty chain)
1. **Research** (15 region agents + 5 global agents) → drafts.
2. **Adversarial verify** (one skeptic per dataset returns the *corrected* data).
3. **Completeness audit** — must reconcile to 120 shrines / 15 towers / 4 Great Fairies. It caught: a duplicated
   desert region (BotW's SW is two tower regions), a duplicate Tera fairy, and a double-filed Mozo Shenno.
4. **Web-verified desert fix** — rebuilt Gerudo Highlands (6) + Gerudo Desert (12) and authoritative per-region
   counts; assembly then found a *121st*: **Shira Gomar**, a Champions' Ballad **DLC** shrine the research agent
   had honestly labeled — excluded (base game only).
5. **`assemble-knowledge.mjs`** applies those fixes and **refuses to write unless it sums to 120 / 15 / 4 with
   zero duplicate names** — the mechanical honesty gate.

## Regenerate
`node build/assemble-knowledge.mjs` → (clean json) → `node build/inline-data.mjs` → `node build/build.mjs`.
